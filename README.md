# keiba

競馬予想を、**必要な情報だけで3段階採点する**ためのリポジトリです。

## 現行ロジック

- `docs/scoring_logic_v2.md`

旧 `scoring_logic_v1.md` は初期案として残しますが、予想には使用しません。

## 基本方針

情報を大量に並べることを目的にしません。
過去に収集した情報は破棄せず、採点項目の根拠として使います。

予想では全頭を次の3段階で採点します。

1. 過去実績点
2. 今回条件点
3. 当日状態点

合計点の高い馬を基本順位とし、その後に1着・2着・3着適性を分けます。

## 残す確認項目

- レース条件
- 出走馬、年齢、斤量、騎手
- 今回に近い過去レース
- 着差、通過順、相手レベル
- コース、距離、馬場、展開、枠
- 馬体重、調教、パドック
- オッズ

取得しただけで順位に影響しない情報は、予想本文へ並べません。

## 採点の役割

### 第1段階：過去実績点

- 今回に近い条件の実績
- 相手レベルと内容
- 近走の安定性・上昇度
- 勝ち切る力
- 年齢・能力低下の減点

### 第2段階：今回条件点

- コース・距離・馬場
- 展開・想定4角位置
- 斤量
- 枠
- 騎手
- 血統（補助のみ）

### 第3段階：当日状態点

- 馬体重
- パドック
- 調教（補助のみ）
- 当日馬場の微調整

## オッズ

オッズは能力点へ入れませんが、無視もしません。
採点完了後に、期待値、券種、買い目、資金配分、見送り判断へ使用します。

## 省略防止

- 全頭の3段階採点が終わるまで印を付けない
- 一部の人気馬だけ詳しく見ない
- 未取得は空欄にする
- 同じ根拠を重複加点しない
- 全頭詳細は採点表へ保存し、予想本文は上位馬と危険馬だけに絞る

## 結果検証

- 確定着順
- 事前点数と順位
- 買い目、投資、払戻、回収率
- 過大評価・過小評価した項目
- 取得漏れ
- 次回の配点変更

不的中は0点とし、結果後に事前採点を書き換えません。
