# keiba

競馬予想を特徴量ベースで評価し、各段階の採点、最終予想、レース後検証を蓄積するためのリポジトリです。

## このリポジトリの目的

競走馬の評価を一度に決めず、役割の異なる複数の段階に分けて分析します。

- 第一段階：過去実績から基礎能力、潜在能力、成長性を評価
- 第二段階：騎手、血統、調教、ペース対応力、位置取りの柔軟性を評価
- 第三段階：当日の天候、馬場、展開、枠と脚質の相互作用、直前オッズを評価
- 最終段階：各段階の結果を統合し、期待値と馬券戦略を決定
- レース後：予想と実際の結果を比較し、評価ロジックを検証

## 基本方針

- 評価根拠は公式資料または保存済みの競走実績に限定する
- 不明な情報を推測で補完しない
- 情報不足を能力不足として扱わない
- 各段階の役割を混在させない
- 全評価が完了する前に最終印や馬券戦略を決めない
- AIが生成した評価結果は `results/` 配下へ保存する
- 元資料や公式ロジックは `docs/` 配下で管理する
- レース結果に合わせて過去の事前採点を上書きしない
- ロジック変更は、将来のレースにも適用可能な一般ルールとして版管理する

## 公式ロジック

新しいレースでは次を使用します。

- `docs/scoring_logic_v4.md`

## 評価段階

### 第一段階：Historical Ability

過去の競走実績のみを使用し、30点満点で評価します。

主な評価項目：

- 類似条件実績
- 実証済みのレース能力
- 着順に表れない潜在能力
- 成長・下降推移
- 勝ち切る力と再現性
- 年齢・性別の歴史補正

オッズ、人気、当日情報、馬券判断は使用しません。

### 第二段階：Supporting Condition Factors

第一段階完了後、次のいずれかを満たす馬を評価します。

- Stage 1が21点以上
- Stage 1が18点以上、Hidden Abilityが2点以上、Race Abilityが6点以上
- Stage 1完成順位の上位6頭

主な評価項目：

- 騎手
- 血統
- 調教
- 通算成績率
- 関連条件での自己ベスト
- ペース対応力
- 位置取りの柔軟性

第二段階ではオッズ、人気、期待値、馬券戦略を使用しません。

### 第三段階：Race-Day Conditions

第一段階と第二段階完了後、30点満点で評価します。

主な評価項目：

- 天候適性
- 当日馬場適性
- 想定ペース
- 枠順と位置取りの相互作用
- 公式当日情報
- 直前オッズ

外枠・内枠を単独で加減点せず、スタート、先行力、脚質、位置取りの柔軟性、最初のコーナーまでの距離、コース形態と組み合わせて評価します。

AI順位と市場順位が大きく異なる場合は、情報漏れ、馬番対応、計算、通過条件を再確認します。ただし、市場順位に合わせるためだけの点数変更は禁止します。

## 主なディレクトリ

```text
docs/       元資料、公式ロジック、評価ルール
results/    AIが生成した各段階の評価結果
    stage1/
    stage2/
    stage3/
    final/
templates/  採点表や出力ファイルのテンプレート
```

## 全体の流れ

```text
Stage 1：過去実績による能力評価
↓
Stage 1全頭完了後にStage 2対象馬を決定
↓
Stage 2：補助条件、ペース対応力、位置取り柔軟性
↓
Stage 3：当日条件、展開、枠と脚質、直前市場
↓
市場との乖離が大きい馬を再確認
↓
3段階のスコアを統合
↓
期待値、券種、買い目、資金配分、見送りを決定
↓
予想結果を保存
↓
レース後に結果検証
```

過去のロジックは検証履歴として残しますが、新しいレースでは最新版の公式ロジックを使用します。
