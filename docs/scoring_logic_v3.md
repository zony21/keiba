# 競馬AI採点ロジック v3.0

## 目的
第一段階を4頭ずつ4回の指示で完了し、処理途中でも同一基準・同一精度を維持する。

## 基本原則
- 馬単位で全項目を採点しない。
- 各回の対象4頭を、評価軸単位で横比較する。
- 必ず「生データ確認 → 特徴量抽出 → 横比較 → 点数変換 → 理由保存」の順で処理する。
- 未取得情報は0点や能力不足へ置き換えず、`未取得`として記録する。
- 抽象語だけの評価は禁止する。着差、通過順、上がり、人気、クラスなどの事実を根拠にする。
- 全16頭完了前に印、買い目、期待値順位を付けない。

# 第一段階：過去実績 30点

## 実行単位
- 第1回：馬番1〜4
- 第2回：馬番5〜8
- 第3回：馬番9〜12
- 第4回：馬番13〜16

各回は必ず対象4頭すべてを完了させる。精度低下を理由に途中停止しない。

## 1. 条件適性 0〜12点
特徴量を次の順で4頭分抽出する。

- 距離適性：0〜3点
- コース・小回り適性：0〜2点
- 右左回り適性：0〜1点
- 馬場適性：0〜1点
- クラス適性：0〜2点
- 頭数・位置・ペース適性：0〜2点
- ハンデ条件適性：0〜1点

直接実績を最優先し、類似条件は補助材料とする。

## 2. レース能力 0〜8点
- 着差：0〜3点
- 相手・勝ち馬レベル：0〜2点
- 通過順とレース内容：0〜1点
- 上がり：0〜1点
- 展開補正：0〜1点

着順だけで判断せず、着差とクラスを優先する。同じ根拠を複数項目へ重複加点しない。

## 3. 能力推移 -6〜6点
年齢は独立評価しない。以下の推移から能力上昇・維持・低下を判定する。

- 条件補正後の着差推移
- 4角位置の推移
- 上がり順位の推移
- 人気と着順の差
- クラス変動を補正した内容推移

不適条件の大敗は分離するが、都合よく除外しない。

## 4. 再現性 0〜4点
- 勝率
- 連対率
- 好走率
- 条件一致時の好走率
- 展開依存度
- 勝ち切る力

近5走だけで母数が不足する場合は、その事実を未取得情報へ記録する。

# 各回の固定処理

## 工程1 生データ確認
過去5走から、競馬場、距離、芝ダート、回り、馬場、クラス、頭数、斤量、着順、着差、人気、通過順、上がり、勝ち馬、ペースを確認する。

## 工程2 特徴量抽出
対象4頭について、同じ特徴量を連続して抽出する。1頭分の全評価を先に完成させない。

## 工程3 横比較
4頭の特徴量表を作り、相対差と固定基準の両方を確認する。

## 工程4 点数変換
特徴量完成後にのみ点数を付ける。

## 工程5 保存
各項目の点数、具体的根拠、未取得情報をCSVと詳細ファイルへ保存する。

# 精度維持ルール
- 前回4頭の過去5走全文を保持し続けず、確定特徴量と点数のみ参照する。
- 2回目以降も配点幅と判定基準を変更しない。
- 「安定」「上昇」「下降」「適性あり」だけの記述は禁止する。
- 情報不足と能力不足を区別する。
- 4回目終了後は全16頭の基準矛盾だけを検査し、理由なく全頭再採点しない。

# 第二段階：今回条件 30点
第一段階完了後に実施する。

- コース・距離・当日馬場：0〜12点
- 展開・想定4角位置：-6〜8点
- 斤量：-4〜4点
- 枠順：-3〜3点
- 騎手：-2〜2点
- 血統：-2〜2点

# 第三段階：当日状態 10点
- 馬体重・推移：-4〜4点
- パドック：-3〜3点
- 調教：-2〜2点
- 当日馬場差：-1〜1点

# 順位・期待値
最終点は3段階の合計。能力順位確定後にのみオッズを参照し、期待値順位、券種、買い目、資金配分、見送りを決定する。

# 結果検証
事前採点は上書きしない。確定着順、投資、払戻、回収率、過小評価、過大評価、取得漏れ、次回修正候補を追記する。不的中は0点として記録する。
