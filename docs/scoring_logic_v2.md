# 競馬AI採点ロジック v3.0

## 目的
全頭を同じ基準で3段階評価し、特徴量から点数を生成して順位を決める。

採点を先に行わず、必ず次の順で処理する。

1. 生データ確認
2. 特徴量抽出
3. 評価軸ごとの横比較
4. 点数変換
5. 全頭統合
6. 印・期待値・買い目決定

未取得情報は空欄または `未取得` とし、推測しない。

# 絶対ルール

## 馬単位採点は禁止

以下の進め方は禁止する。

```text
1番の全項目を採点
2番の全項目を採点
3番の全項目を採点
```

各回の対象馬について、評価軸単位で横方向に処理する。

```text
全対象馬の距離適性
全対象馬のコース適性
全対象馬の右左回り適性
全対象馬の小回り適性
...
```

## 第一段階は4回で完了する

出走頭数を原則4分割し、4回の指示で第一段階を完了させる。
16頭の場合は次の固定単位とする。

- 第1回：1〜4番
- 第2回：5〜8番
- 第3回：9〜12番
- 第4回：13〜16番

各回で対象馬の第一段階を必ず完了する。
途中停止や「精度を保てないため続行不可」は禁止する。

# 第1段階：過去実績 30点

第1段階は次の4項目で構成する。

1. 条件適性：0〜12点
2. レース能力：0〜8点
3. 能力推移：-6〜6点
4. 再現性：0〜4点

年齢は独立評価しない。
能力低下は着差、位置取り、上がり、条件補正後の内容推移から能力推移へ反映する。

## 1. 条件適性：0〜12点

点数を付ける前に、以下の特徴量を抽出する。

- 距離実績
- 競馬場実績
- 右回り・左回り実績
- 小回り・大回り実績
- 馬場状態別実績
- クラス別実績
- 頭数別実績
- ハンデ戦実績
- ペース適性
- 今回に近い条件での好走数

配点目安：

- 距離：0〜3点
- コース・小回り：0〜2点
- 右左回り：0〜1点
- 馬場：0〜1点
- クラス：0〜2点
- 頭数・展開・ペース：0〜2点
- ハンデ条件：0〜1点

## 2. レース能力：0〜8点

抽出する特徴量：

- 平均着差
- 最小着差
- 重賞・オープンでの着差
- 勝ち馬のレベル
- 相手レベル
- 通過順
- 上がり順位
- 展開不利・有利
- 着順より内容が良かったレース数

配点目安：

- 着差：0〜3点
- 相手・勝ち馬レベル：0〜2点
- 通過順と内容：0〜1点
- 上がり：0〜1点
- 展開補正：0〜1点

## 3. 能力推移：-6〜6点

年齢ではなく、条件差を補正した近走内容の変化を見る。

抽出する特徴量：

- 着差推移
- 4角位置推移
- 上がり順位推移
- 人気と着順の差
- クラス補正後の内容推移
- 距離・馬場・コース補正後の指数推移

抽象語だけで判定してはいけない。
「上昇」「下降」「安定」と記載する場合は、必ず数値列または具体的なレース内容を併記する。

## 4. 再現性：0〜4点

抽出する特徴量：

- 勝率
- 連対率
- 3着内率
- 条件一致時好走率
- 大敗率
- 展開依存度
- 脚質依存度
- 勝ち切る力

# 各回の固定処理

1回の指示で次をすべて完了する。

1. 対象馬の生データ確認
2. 特徴量抽出
3. 評価軸ごとの横比較
4. 条件適性点
5. レース能力点
6. 能力推移点
7. 再現性点
8. 第1段階合計
9. 採点理由
10. 未取得情報
11. CSV保存用データ

情報不足の場合も、取得済み情報の範囲で評価を完了し、不足項目を明記する。
情報不足を能力不足として0点評価してはいけない。

# 精度維持ルール

- 生データから特徴量を抽出する前に点数を付けない。
- 特徴量を横比較する前に理由文を完成させない。
- 同じ根拠を複数項目へ重複加点しない。
- 前回対象馬の過去走全文を保持し続けない。
- 前回までの確定値は特徴量CSVと採点CSVだけを参照する。
- 後半の馬ほど説明や確認項目を減らしてはいけない。
- 未取得情報を推測で補完しない。
- 4回終了まで印、期待値順位、買い目を付けない。

# 第1段階終了時の整合性検査

4回目終了後に全頭を統合し、以下だけを確認する。

- 同一特徴量に対する配点基準の不一致
- 情報不足と能力不足の混同
- 同一根拠の重複加点
- 合計式の誤り
- 前半と後半で説明密度が異なっていないか

原則として全頭を最初から再採点しない。
矛盾が見つかった項目だけ修正する。

# 第2段階：今回条件 30点

1. コース・距離・馬場適性：0〜12点
2. 展開・想定4角位置：-6〜8点
3. 斤量：-4〜4点
4. 枠順：-3〜3点
5. 騎手：-2〜2点
6. 血統：-2〜2点

直接実績を最優先する。
騎手と血統は補助評価とし、直接実績を逆転させない。

# 第3段階：当日状態 10点

1. 馬体重・推移：-4〜4点
2. パドック：-3〜3点
3. 調教：-2〜2点
4. 当日馬場差：-1〜1点

調教だけで能力順位を変えない。

# 順位・オッズ・買い目

最終点は以下とする。

```text
最終点 = 第1段階 + 第2段階 + 第3段階
```

- 全頭採点後にのみ印を付ける。
- 1着候補は勝ち切る力を重視する。
- 2着候補は安定性を重視する。
- 3着候補は条件特化、展開利、人気薄の条件一致を重視する。
- オッズは能力点へ入れない。
- 採点完了後に能力順位と期待値順位を分離する。
- 人気薄でも条件一致度が高い馬は期待値順位で上位へ置く。

# 保存構造

レースごとに `races/YYYY/YYYYMMDD_レース名/` を作成する。

- `docs/`：ユーザー提供資料
- `feature_condition.csv`：条件適性特徴量
- `feature_ability.csv`：レース能力特徴量
- `feature_trend.csv`：能力推移特徴量
- `feature_repeatability.csv`：再現性特徴量
- `score.csv`：点数と確定理由
- `prediction.md`：印、期待値、買い目
- `review.md`：結果検証

採点後や結果確定後も、事前データを削除・上書きしない。
修正が必要な場合は変更理由を記録する。

# 結果検証

- 確定着順
- 事前特徴量
- 事前点数と順位
- 買い目、投資、払戻、回収率
- 1〜3着馬の過小評価項目
- 上位予想馬の過大評価項目
- 取得漏れ
- 次回変更する特徴量または配点

不的中は0点として記録する。
結果後に事前採点を書き換えない。
