# 競馬AI採点ロジック v2.1

## 目的
全頭を同じ基準で3段階採点し、合計点で順位を決める。
情報は採点根拠として使い、確認項目を増やしすぎない。

## 使用する情報
- レース条件：競馬場、距離、芝・ダート、馬場、クラス、斤量条件
- 出走馬：馬番、馬名、年齢、斤量、騎手
- 過去成績：今回に近いレース、着差、通過順、相手レベル、近走推移
- 今回条件：コース適性、距離適性、展開、枠
- 当日状態：馬体重、調教、パドック
- オッズ：採点後の期待値判断に使用

未取得情報は空欄とし、推測しない。

# 第1段階：過去実績 30点

1. 今回に近い条件の実績：0〜12点
2. 相手レベル・着差・レース内容：0〜8点
3. 近走の維持・上昇・低下：-6〜6点
4. 勝ち切る力・安定性：0〜4点
5. 年齢による能力低下：0〜-4点

年齢だけでは減点しない。着差拡大、位置取り悪化、反応低下などが確認できた場合のみ減点する。

# 第2段階：今回条件 30点

1. コース・距離・馬場適性：0〜12点
2. 展開・想定4角位置：-6〜8点
3. 斤量：-4〜4点
4. 枠順：-3〜3点
5. 騎手：-2〜2点
6. 血統：-2〜2点

直接実績を最優先する。騎手と血統は補助評価とし、直接実績を逆転させない。

# 第3段階：当日状態 10点

1. 馬体重・推移：-4〜4点
2. パドック：-3〜3点
3. 調教：-2〜2点
4. 当日馬場差：-1〜1点

調教だけで能力順位を変えない。

# 順位決定

最終点 = 第1段階 + 第2段階 + 第3段階

- 全頭を採点してから印を付ける
- 同点または2点以内だけ、今回に近い実績、展開、斤量、当日状態の順で比較する
- 1着候補は勝ち切る力、2着候補は安定性、3着候補は条件特化・展開利も確認する

# オッズ

オッズは能力点に入れないが無視しない。
採点完了後に、人気・不人気の理由が今回も有効か確認し、期待値順位、券種、買い目、資金配分、見送りへ反映する。

# 省略防止

- 全頭採点が終わるまで印を付けない
- 一部の馬だけ詳しく見ない
- 同じ根拠を重複加点しない
- 予想本文は上位馬・危険馬・買い目だけに絞る
- 全頭の点数と根拠はCSVへ保存する

# 結果検証

- 確定着順
- 事前点数と順位
- 買い目、投資、払戻、回収率
- 1〜3着馬の過小評価項目
- 上位予想馬の過大評価項目
- 取得漏れ
- 次回変更する配点

不的中は0点。結果後に事前採点を書き換えない。
